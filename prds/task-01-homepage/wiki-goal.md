### Cursor AI辅助开发实践 Wiki 结构（最终版）

最终结构只有 **4 大分类**，页面总数进一步压缩到 **13 个**，逻辑流更清晰：
1. 基础与上手  
2. 工程化、技术栈与适配（整合后放在第二位）  
3. 核心功能与配置  
4. 提示工程（最重头的部分放在最后，便于读者先打好工程基础再深入提示）

这样阅读路径更自然：上手 → 工程化与选栈（保障代码可靠） → Rules 配置 → 提示工程（发挥最大威力）。

#### Wiki 整体结构

**首页（Home / 主页）**
- Wiki 简介与动机
- Cursor 核心优势一览
- 快速导航卡片（4 大分类入口）
- **参考速查区**（模型对比表格、完整快捷键清单、推荐外部资源、术语表）

**分类目录（Sidebar / 顶部导航）**
1. 基础与上手  
2. 工程化、技术栈与适配  
3. 核心功能与配置  
4. 提示工程  

#### 详细页面清单（共 13 个页面）

##### 1. 基础与上手
- **快速上手 Cursor**  
  界面、核心功能（Chat、Composer、Cmd+K、@引用、Apply/Diff、Inline Edit）、快捷键、安装配置、项目索引优化、初学者常见坑。  
  → 结尾整合：完整快捷键表、术语表。
- **模型选择与成本优化**  
  主流模型对比表格、场景选型指南、实测数据、成本控制技巧。  
  → 结尾整合：模型选择常见陷阱与最佳实践。

##### 2. 工程化、技术栈与适配（整合后的新分类）
- **AI 时代技术栈选择与适配**  
  热门技术栈分析（React + TS + Zod、NestJS/FastAPI、Next.js 15 等为什么更 AI 友好）、选型标准、monorepo 索引策略、框架特定优化技巧。  
  → 嵌入跨技术栈集成案例的关键技巧作为实际演示。  
  → 结尾整合：技术栈选型与适配的 Do/Don't。
- **工程化实践：Lint、Format 与 CI/CD**  
  Cursor Rules 与 eslint/prettier 联动、AI 生成代码后的格式化、GitHub Actions 集成、双重审查机制、质量问题防范。  
  → 结尾整合：工程化常见陷阱与最佳实践。
- **自动化测试的 AI 辅助**  
  生成单元/集成/e2e 测试、测试驱动开发的 AI 变种、先写测试再实现、测试覆盖率审查。  
  → 嵌入复杂 bug 排查与性能优化案例中涉及测试的部分作为演示。

##### 3. 核心功能与配置
- **Cursor Rules 制定指南**（重点页面）  
  Rules 作用、编写原则、进阶技巧（条件规则、动态引用、与工具链联动）。  
  → 嵌入 2-3 个简要示例，并链接到【Rules 示例合集】。  
  → 结尾整合：Rules 最佳实践与常见坑（如规则过多影响性能）。
- **Rules 示例合集**  
  分技术栈分类（React + TS、NestJS、FastAPI、Next.js、monorepo 等），每个示例可复制 + 适用场景说明。  
  → 嵌入重构案例中涉及 Rules 的实际应用片段。

##### 4. 提示工程（最核心模块）
- **提示工程核心原则**  
  基础到进阶（清晰、上下文、迭代、角色扮演、结构化）。  
  → 结尾整合：提示工程 Do and Don't。
- **结构化 PRD 提示词实战**（重点页面）  
  从模糊需求 → 技术方案 → 任务拆解 → 代码生成的完整链条、模板设计、多轮迭代技巧。  
  → **完整嵌入从 PRD 到新功能上线案例**（含全链路 Prompt 记录）。  
  → 所有模板链接到【Prompt 模板库】。
- **Prompt 模板库**  
  分类存放（PRD 类、新功能、重构、调试/优化、测试生成、文档生成），每个模板带说明、变量、推荐模型、可直接复制。  
  → **嵌入遗留代码重构案例和复杂 bug 排查案例的关键 Prompt** 作为真实示例。

#### 最终版优势总结
- **极简 4 大类、13 页面**：内容高度浓缩，无任何独立案例页、最佳实践页、参考页，所有内容就近融入。
- **逻辑流畅**：先上手 + 模型 → 建立可靠工程体系和技术栈 → 配置 Rules → 深入提示工程，形成“基础 → 保障 → 配置 → 爆发”的进阶路径。
- **实战性强**：所有原案例内容已无缝嵌入相关页面，读者学到技巧时立刻看到真实应用和避坑建议。
- **易维护易检索**：模板库和 Rules 合集仍独立，便于持续补充和快速查找。
