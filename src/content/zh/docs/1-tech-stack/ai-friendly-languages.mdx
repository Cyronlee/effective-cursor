# AI 时代的宠儿

> 什么样的语言和技术最适合 AI 辅助开发？

AI 模型是通过大量代码语料训练而来的，因此不同的编程语言和框架对 AI 的"友好程度"是不同的。了解这些差异可以帮助你选择更适合 AI 辅助开发的技术栈。

## 强类型语言优先

强类型语言对 AI 更加友好，原因如下：

- **类型信息提供上下文** - AI 可以从类型推断变量用途和函数意图
- **更少的歧义** - 类型约束减少了代码理解的歧义
- **更好的代码补全** - IDE 和 AI 都能提供更精准的建议

**推荐**: TypeScript、Go、Rust、Java、C#

## 结构清晰的代码

AI 更容易理解和生成结构化的代码：

- 清晰的目录结构
- 一致的命名规范
- 模块化的组织方式

## 语义化设计

语义化的代码让 AI 更容易理解意图：

```typescript
// ✅ 语义化 - AI 容易理解
function calculateOrderTotal(items: CartItem[]): Money {
  return items.reduce((total, item) => total.add(item.price), Money.zero())
}

// ❌ 非语义化 - AI 难以推断意图
function calc(arr: any[]): number {
  return arr.reduce((a, b) => a + b.p, 0)
}
```

## Markdown 的 AI 友好性

Markdown 在 AI 辅助开发中扮演重要角色：

- **文档即代码** - AI 可以同时理解文档和代码的关系
- **结构化表达** - 标题、列表、代码块提供清晰的结构
- **Prompt 友好** - 大多数 AI 工具都原生支持 Markdown 格式

在项目中善用 Markdown 可以：
- 编写清晰的 README 和文档
- 使用注释和 JSDoc 描述代码意图
- 创建结构化的 Prompt 模板

## 成熟的生态系统

选择有成熟生态的技术栈：

- **丰富的训练数据** - AI 见过更多相关代码
- **社区最佳实践** - AI 能给出更符合惯例的建议
- **完善的工具链** - 更好的开发体验
