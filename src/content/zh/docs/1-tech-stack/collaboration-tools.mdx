# 协作工具

> 提升团队效率的 AI 友好工具推荐

除了核心的开发技术栈，选择合适的协作工具同样可以提升 AI 辅助开发的效率。

## Git 客户端

### Fork

- 可视化 Git 操作
- 直观的分支管理
- 冲突解决友好

### 其他选择

- **GitKraken** - 功能丰富的 Git 客户端
- **Sourcetree** - Atlassian 出品的免费客户端
- **命令行** - 最直接的方式，AI 可以直接生成 Git 命令

## API 测试工具

### Bruno

- 开源、本地优先
- 支持 Git 版本控制
- 无需云账户

```
collections/
├── api/
│   ├── users/
│   │   ├── get-user.bru
│   │   └── create-user.bru
│   └── orders/
│       └── list-orders.bru
└── environments/
    ├── local.bru
    └── production.bru
```

### 其他选择

- **Postman** - 功能全面的 API 平台
- **Insomnia** - 轻量级 REST 客户端
- **HTTPie** - 命令行 HTTP 客户端

## 文档工具

### Docusaurus

- React 驱动的文档站点
- Markdown/MDX 支持
- 搜索、版本控制内置

### 其他选择

- **VitePress** - Vue 驱动，轻量快速
- **Nextra** - Next.js 生态的文档框架
- **Notion** - 团队知识库

## 设计协作

### Figma

- 设计稿可直接导出代码
- AI 工具可以读取设计稿
- 团队协作便捷

## 项目管理

### Linear

- 现代化的 Issue 追踪
- 键盘快捷键友好
- 与开发工作流集成

### 其他选择

- **GitHub Projects** - 与代码库紧密集成
- **Jira** - 企业级项目管理
- **Notion** - 灵活的项目管理

## AI 辅助开发的协作最佳实践

1. **文档即代码** - 使用 Markdown 编写文档，与代码一起版本控制
2. **API 规范优先** - OpenAPI/Swagger 规范让 AI 更容易理解接口
3. **统一编码规范** - ESLint、Prettier 配置共享，AI 生成代码更一致
4. **结构化的 Commit** - 使用 Conventional Commits，便于 AI 理解变更历史

## 总结

选择正确的协作工具可以：
- 减少团队沟通成本
- 让 AI 更容易理解项目上下文
- 提升整体开发效率

好的工具链是高效 AI 辅助开发的基础。
